## Google Search Console Setup

### Initial Setup Steps

#### 1. Verify Domain Ownership
**Recommended Method: DNS Verification** (proves ownership of entire domain including subdomains)

1. Go to [Google Search Console](https://search.google.com/search-console)
2. Click "Add Property"
3. Select "Domain" property type
4. Enter: `agentbio.net`
5. Copy the TXT record provided
6. Add to DNS settings:
   ```
   Type: TXT
   Name: @
   Value: google-site-verification=XXXXXXXXXXXXXXXXXXXX
   TTL: 3600
   ```
7. Wait 24-48 hours for DNS propagation
8. Return to Search Console and click "Verify"

**Alternative Method: HTML Tag Verification**
```html
<!-- Add to <head> section of index.html -->
<meta name="google-site-verification" content="XXXXXXXXXXXXXXXXXXXX" />
```

#### 2. Submit Property for All URL Prefixes
After domain verification, add these properties for detailed tracking:
- `https://agentbio.net`
- `https://www.agentbio.net` (if applicable)

#### 3. Configure Settings
Navigate to Settings → Users and permissions:
- Add team members with appropriate access levels:
  - **Owner**: Full control (1-2 people)
  - **Full User**: Can view all data and take some actions (developers, SEO team)
  - **Restricted User**: Can view most data (marketing team)

#### 4. Link Google Analytics
Settings → Associations → Link to Google Analytics property
- Enables cross-platform data analysis
- Shares audience data between platforms

## Sitemap Submission & Monitoring

### Sitemap Structure

Our site uses a **dynamic sitemap** generated by Supabase Edge Function:

**Sitemap URL**: `https://agentbio.net/sitemap.xml`

**Current Coverage**:
- Homepage
- Main pages (Pricing, Blog, Privacy, Terms, DMCA, Acceptable Use)
- Agent profiles (dynamic, from database)
- Blog articles (dynamic, from database)
- Custom pages (dynamic, from database)
- Blog category pages

### Submitting Sitemaps to Search Engines

#### Google Search Console
1. Navigate to "Sitemaps" in left sidebar
2. Enter sitemap URL: `https://agentbio.net/sitemap.xml`
3. Click "Submit"
4. Status should change to "Success" within 24 hours

#### Bing Webmaster Tools
1. Go to [Bing Webmaster Tools](https://www.bing.com/webmasters)
2. Add site and verify ownership (similar to Google)
3. Navigate to "Sitemaps" → "Submit Sitemap"
4. Enter: `https://agentbio.net/sitemap.xml`
5. Click "Submit"

#### robots.txt Configuration
Ensure robots.txt includes sitemap reference:
```
# /public/robots.txt
User-agent: *
Allow: /

# Sitemaps
Sitemap: https://agentbio.net/sitemap.xml

# Disallow private areas
Disallow: /dashboard
Disallow: /admin
Disallow: /auth
Disallow: /api
```

### Monitoring Sitemap Health

#### Daily Checks
- **Status**: Ensure sitemap is "Success" not "Couldn't fetch" or "Has errors"
- **Discovered URLs**: Track count of discovered pages
- **Indexed URLs**: Track count of indexed pages

#### Key Metrics to Track
| Metric | Location | Good Target |
|--------|----------|-------------|
| Sitemap Status | Search Console → Sitemaps | "Success" |
| Submitted URLs | Sitemap details | Matches actual page count |
| Discovered URLs | Sitemap details | 90%+ of submitted |
| Indexed URLs | Sitemap details | 80%+ of submitted |

#### Common Sitemap Issues & Fixes

**Issue**: "Couldn't fetch"
- **Cause**: Sitemap URL returns 404 or server error
- **Fix**:
  - Verify sitemap accessible: `curl -I https://agentbio.net/sitemap.xml`
  - Check Edge Function logs in Supabase
  - Ensure no authentication required

**Issue**: "Sitemap is HTML page"
- **Cause**: Server returning HTML instead of XML
- **Fix**:
  - Check Content-Type header: `Content-Type: application/xml`
  - Verify Edge Function corsHeaders configuration

**Issue**: Low indexed percentage
- **Cause**: Pages blocked by robots.txt, noindex tags, or quality issues
- **Fix**:
  - Check Coverage report for exclusion reasons
  - Review noindex tags on affected pages
  - Improve content quality on thin pages

## Crawl Rate & Coverage Tracking

### Understanding Coverage Reports

#### Coverage Report Location
Google Search Console → Coverage (or "Pages" in new interface)

#### Page Status Categories

**✅ Indexed (Valid)**
- Pages successfully crawled and indexed
- **Target**: 80-95% of submitted pages
- **Action**: Monitor for sudden drops

**⚠️ Excluded (Valid)**
- Pages intentionally excluded (noindex, canonical, etc.)
- **Target**: Should match intentional exclusions
- **Action**: Verify these are correct

**❌ Error**
- Server errors (5xx), not found (404), unauthorized
- **Target**: 0%
- **Action**: Fix immediately

**⏸️ Crawled - not indexed**
- Crawled but not added to index (quality issues)
- **Target**: < 5%
- **Action**: Improve content quality or remove

### Crawl Stats Monitoring

#### Crawl Stats Location
Google Search Console → Settings → Crawl Stats

#### Key Metrics
- **Total crawl requests**: Number of times Googlebot requested pages
  - **Good**: Increasing trend over time
  - **Bad**: Sudden drops indicate crawling issues

- **Total download size (KB)**: Data transferred during crawling
  - **Target**: Stable or decreasing (indicates better optimization)

- **Average response time (ms)**: Server response speed
  - **Target**: < 500ms
  - **Warning**: > 1000ms affects crawl budget

- **Crawl by response code**: Distribution of HTTP responses
  - **Target**: 90%+ should be 200 (success)
  - **Monitor**: 404s, 301s, 5xx errors

- **Crawl by file type**: Distribution of crawled resources
  - HTML should be majority (60-80%)
  - Monitor excessive CSS/JS/image crawling

- **Crawl by purpose**: Why Googlebot crawled
  - Discovery: New pages found via links/sitemap
  - Refresh: Recrawling existing pages

#### Optimizing Crawl Budget

**For Small Sites (< 10,000 pages)**
- Crawl budget is typically not an issue
- Focus on server performance and quality

**For Growing Sites (> 10,000 pages)**
1. **Fix Server Errors**: Priority #1
2. **Improve Response Times**: Target < 500ms
3. **Update Sitemaps Frequently**: Prioritizes important pages
4. **Use Robots.txt Wisely**: Block unnecessary crawling
5. **Minimize Redirects**: Each redirect wastes crawl budget
6. **Eliminate Duplicate Content**: Use canonicals

### Coverage Tracking Workflow

#### Weekly Review (15 minutes)
1. Check Coverage report for new errors
2. Review "Discovered - currently not indexed" pages
3. Verify submitted vs. indexed count
4. Check for sudden drops in indexed pages

#### Monthly Deep Dive (1 hour)
1. Analyze trends in indexed pages
2. Review excluded pages - verify correctness
3. Check crawl stats for anomalies
4. Investigate pages "Crawled - not indexed"
5. Update sitemap with new pages

#### Quarterly Audit (2-3 hours)
1. Full site crawl with Screaming Frog or similar
2. Compare internal crawl vs. Search Console data
3. Identify orphan pages (not in sitemap or internal links)
4. Fix technical SEO issues found
5. Document changes and impact

## Performance Metrics

### Core Web Vitals

Google uses these as ranking factors:

#### 1. Largest Contentful Paint (LCP)
**Measures**: Time to render largest visible element
- **Good**: < 2.5 seconds
- **Needs Improvement**: 2.5 - 4.0 seconds
- **Poor**: > 4.0 seconds

**Common causes of slow LCP**:
- Unoptimized hero images
- Slow server response times
- Render-blocking JavaScript/CSS
- Client-side rendering delays

**Fixes**:
- Optimize and compress hero images
- Use WebP format with proper sizing
- Implement CDN for faster delivery
- Preload critical resources
- Use lazy loading for below-fold images

#### 2. First Input Delay (FID) / Interaction to Next Paint (INP)
**Measures**: Responsiveness to user interactions
- **Good**: < 100ms (FID) / < 200ms (INP)
- **Needs Improvement**: 100-300ms / 200-500ms
- **Poor**: > 300ms / > 500ms

**Common causes**:
- Heavy JavaScript execution
- Long-running tasks blocking main thread
- Large bundle sizes

**Fixes**:
- Code splitting with lazy loading
- Defer non-critical JavaScript
- Use web workers for heavy computation
- Optimize React re-renders
- Minimize third-party scripts

#### 3. Cumulative Layout Shift (CLS)
**Measures**: Visual stability (unexpected layout shifts)
- **Good**: < 0.1
- **Needs Improvement**: 0.1 - 0.25
- **Poor**: > 0.25

**Common causes**:
- Images without dimensions
- Ads/embeds without reserved space
- Dynamic content injection
- Web fonts causing FOIT/FOUT

**Fixes**:
- Set explicit width/height on images
- Use aspect-ratio CSS property
- Reserve space for dynamic content
- Use font-display: optional or swap
- Avoid inserting content above existing content

### Monitoring Core Web Vitals

#### Google Search Console
Location: Experience → Core Web Vitals
- Shows real-world data from Chrome users
- Grouped by device (Mobile/Desktop)
- Classification: Good, Needs Improvement, Poor URLs

#### PageSpeed Insights
URL: https://pagespeed.web.dev/
- Provides lab data (simulated) and field data (real users)
- Detailed recommendations for improvements
- Run weekly for key pages

#### Chrome UX Report (CrUX)
- Real user data from Chrome browsers
- Available via BigQuery for advanced analysis
- 28-day rolling average

### Other Critical Performance Metrics

#### Server Response Time (TTFB)
- **Target**: < 200ms
- **Monitor**: Via Google Search Console → Crawl Stats
- **Impacts**: All other performance metrics

#### Page Load Time
- **Target**: < 3 seconds on 4G
- **Monitor**: Via Google Analytics, WebPageTest
- **Impacts**: User engagement and conversions

#### Mobile Performance
- **Target**: Mobile score within 10 points of desktop
- **Monitor**: PageSpeed Insights mobile vs desktop
- **Critical**: 60%+ of traffic is mobile

## Analytics Integration

### Google Analytics 4 (GA4) Setup

#### 1. Create GA4 Property
1. Go to [Google Analytics](https://analytics.google.com/)
2. Click "Admin" → "Create Property"
3. Enter property details
4. Create "Web" data stream
5. Copy Measurement ID: `G-XXXXXXXXXX`

#### 2. Install GA4 Tag
```typescript
// src/lib/analytics.ts
import ReactGA from 'react-ga4';

const MEASUREMENT_ID = import.meta.env.VITE_GA_MEASUREMENT_ID;

export const initGA = () => {
  if (MEASUREMENT_ID) {
    ReactGA.initialize(MEASUREMENT_ID);
  }
};

export const logPageView = (path: string) => {
  ReactGA.send({ hitType: "pageview", page: path });
};

export const logEvent = (category: string, action: string, label?: string) => {
  ReactGA.event({ category, action, label });
};
```

```typescript
// src/App.tsx
import { useEffect } from 'react';
import { useLocation } from 'react-router-dom';
import { initGA, logPageView } from './lib/analytics';

function App() {
  const location = useLocation();

  useEffect(() => {
    initGA();
  }, []);

  useEffect(() => {
    logPageView(location.pathname + location.search);
  }, [location]);

  // ... rest of component
}
```

## Tracking SEO ROI

### Key Performance Indicators (KPIs)

#### Visibility Metrics
1. **Organic Traffic Growth**: Month-over-month increase
   - **Target**: 10-15% monthly growth
   - **Formula**: (Current month - Last month) / Last month × 100

2. **Keyword Rankings**: Position for target keywords
   - **Target**: Top 10 for primary keywords
   - **Track**: 20-30 core keywords

3. **Click-Through Rate (CTR)**: From search results
   - **Target**: > 2% average (varies by position)
   - **Benchmark**: Position 1 = ~30%, Position 10 = ~2%

#### Engagement Metrics
1. **Organic Bounce Rate**: Single-page sessions from search
   - **Target**: < 60%
   - **Low bounce = relevant traffic**

2. **Pages per Session**: Content depth
   - **Target**: > 2.5 pages
   - **Higher = better engagement**

3. **Average Session Duration**: Time on site
   - **Target**: > 2 minutes
   - **Higher = quality traffic**

#### Conversion Metrics
1. **Organic Conversion Rate**: Leads from organic traffic
   - **Target**: 2-4% for real estate
   - **Formula**: Conversions / Organic Sessions × 100

2. **Cost per Acquisition (CPA)**: SEO investment / conversions
   - **Target**: Lower than paid search CPA
   - **Track**: Monthly SEO spend vs. organic leads

3. **Customer Lifetime Value (CLV)**: Long-term value of organic leads
   - **Target**: Higher than paid channels
   - **Track**: Quality of organic vs paid leads
